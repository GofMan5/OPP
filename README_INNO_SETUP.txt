Инструкция по созданию установщика с помощью Inno Setup
=================================================

1. Установка Inno Setup
----------------------
1. Скачайте Inno Setup с официального сайта: https://jrsoftware.org/isdl.php
2. Установите программу, следуя инструкциям мастера установки.

2. Подготовка файлов
------------------
1. Соберите ваш проект в режиме Release.
2. Подготовьте иконку для установщика (App.ico) и положите её в ту же директорию, где находится скрипт GofMan3_Setup.iss.

3. Настройка параметров скрипта
-----------------------------
Откройте файл GofMan3_Setup.iss и отредактируйте следующие параметры:

- В начале файла:
  * MyAppName - имя приложения
  * MyAppVersion - версия приложения
  * MyAppPublisher - издатель
  * MyAppURL - ссылка на сайт или GitHub репозиторий
  * MyAppExeName - имя исполняемого файла

- В секции [Files]:
  * Source - путь к скомпилированным файлам вашего приложения
  
Если нужна дополнительная настройка, измените другие параметры в соответствии с потребностями.

4. Компиляция установщика
-----------------------
1. Запустите Inno Setup Compiler.
2. Откройте файл GofMan3_Setup.iss.
3. Нажмите Ctrl+F9 или выберите Build > Compile из меню.
4. После успешной компиляции установщик будет создан в директории Output.

5. Тестирование установщика
-------------------------
Обязательно протестируйте установщик на чистой системе или в виртуальной машине:
1. Убедитесь, что все файлы приложения корректно устанавливаются.
2. Проверьте создание ярлыков в меню Пуск и на рабочем столе.
3. Убедитесь, что приложение запускается после установки.
4. Проверьте функцию удаления приложения через Панель управления.

6. Дополнительные функции скрипта
------------------------------
- Скрипт автоматически проверяет наличие .NET Framework 4.8. 
- Поддерживает автоматическое обновление (удаляет старую версию перед установкой новой).
- Создает ярлыки в меню Пуск и на рабочем столе (по выбору пользователя).
- Поддерживает русский язык интерфейса установщика.

7. Возможные проблемы
------------------
- Если ваше приложение требует других версий .NET Framework, измените код проверки в функции IsDotNetDetected.
- При изменении структуры файлов приложения обновите секцию [Files].
- Если требуются дополнительные задачи установки (например, регистрация компонентов), добавьте их в скрипт.

8. Создание новой версии
---------------------
При выпуске новой версии приложения:
1. Обновите значение MyAppVersion в начале скрипта.
2. Скомпилируйте новую версию установщика.
